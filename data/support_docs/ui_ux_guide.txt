1. DESIGN PHILOSOPHY
-------------------
The E-Shop Checkout interface follows modern, clean design principles:
- Minimalist approach with focus on usability
- Clear visual hierarchy
- Immediate user feedback for all actions
- Progressive disclosure of information
- Mobile-first responsive design


2. COLOR PALETTE
----------------
PRIMARY COLORS:
- Brand Purple: #667eea (primary actions, headers)
- Dark Purple: #764ba2 (gradients, accents)
- Success Green: #28a745 (confirmations, pay button)

SEMANTIC COLORS:
- Error Red: #dc3545 (validation errors, remove buttons)
- Warning Yellow: #ffc107 (discount button, alerts)
- Info Blue: #667eea (informational elements)

NEUTRAL COLORS:
- White: #ffffff (backgrounds, cards)
- Light Gray: #f8f9fa (section backgrounds)
- Medium Gray: #dee2e6 (borders, dividers)
- Dark Gray: #495057 (text, labels)
- Muted Gray: #6c757d (secondary text)


3. TYPOGRAPHY
-------------
FONT FAMILY: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif

FONT SIZES:
- Page Header (h1): 2.5em (40px)
- Section Headers (h2): 1.5em (24px)
- Product Titles (h3): Default size
- Body Text: 1em (16px)
- Button Text: 1.1em - 1.3em depending on importance
- Small Text: 0.9em (14px) for descriptions

FONT WEIGHTS:
- Regular: 400 (body text)
- Semi-bold: 600 (labels, emphasis)
- Bold: 700 (prices, totals, buttons)


4. FORM VALIDATION DISPLAY
---------------------------
VALIDATION STATES:

Default State:
- Border: 2px solid #dee2e6 (medium gray)
- Background: white
- No error message visible

Focus State:
- Border: 2px solid #667eea (brand purple)
- Background: white
- Outline: none

Error State:
- Border: 2px solid #dc3545 (error red)
- Background: white
- Error message displayed below field
- Error message color: #dc3545 (error red)
- Error message font-size: 0.9em (14px)

CRITICAL RULE: Form validation errors MUST be displayed in RED TEXT below the invalid field.

VALIDATION TIMING:
- Validation triggers when user clicks "Pay Now" button
- Do NOT validate on field blur (only on form submission)
- Error messages appear immediately upon validation failure
- Error messages disappear when field becomes valid

9. ACCESSIBILITY GUIDELINES
----------------------------

CONTRAST REQUIREMENTS:
- All text must meet WCAG AA standards (4.5:1 ratio minimum)
- Error messages use high contrast red (#dc3545 on white)
- Success messages use high contrast green (#155724 on light green background)

KEYBOARD NAVIGATION:
- All interactive elements must be keyboard accessible
- Tab order follows logical flow
- Focus states clearly visible (purple border)

LABELS AND DESCRIPTIONS:
- All form fields have visible labels
- Radio buttons have descriptive text
- Required fields marked with asterisk (*)

SCREEN READER SUPPORT:
- Use semantic HTML elements
- Provide alt text for icons (if using images)
- Error messages associated with form fields


10. RESPONSIVE BEHAVIOR
-----------------------

BREAKPOINTS:
- Mobile: < 768px
- Desktop: >= 768px

MOBILE ADAPTATIONS:
- Two-column layout becomes single column
- Maintain readability and touch targets
- Buttons remain full-width for easy tapping

GRID LAYOUT:
- Desktop: 2-column grid (products/cart left, form/payment right)
- Mobile: Single column, stacked sections
- Breakpoint: 968px (changes to single column below this)
